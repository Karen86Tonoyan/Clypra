# Security Policy

## Supported Versions

We release patches for security vulnerabilities. Currently supported versions:

| Version | Supported          |
| ------- | ------------------ |
| 0.1.x   | :white_check_mark: |

## Reporting a Vulnerability

We take the security of Clypra seriously. If you believe you have found a security vulnerability, please report it to us as described below.

### Please do NOT:

- Open a public GitHub issue for security vulnerabilities
- Disclose the vulnerability publicly before it has been addressed

### Please DO:

1. **Email us directly** at [your-security-email@example.com] with:
   - Description of the vulnerability
   - Steps to reproduce
   - Potential impact
   - Suggested fix (if any)

2. **Allow time for a fix** - We will:
   - Acknowledge receipt within 48 hours
   - Provide an estimated timeline for a fix
   - Notify you when the vulnerability is fixed
   - Credit you in the security advisory (if desired)

### What to expect:

- We will investigate all legitimate reports
- We will keep you informed of our progress
- We will work to fix verified vulnerabilities promptly
- We will publicly disclose the vulnerability after a fix is released

## Security Best Practices

When using Clypra:

- Keep your installation up to date
- Only import media files from trusted sources
- Be cautious when using third-party plugins (when available)
- Review file permissions before granting access

## Security Updates

Security updates will be released as patch versions and announced via:

- GitHub Security Advisories
- Release notes
- Project README

Thank you for helping keep Clypra and its users safe!
